# 03 · Frontend Chart Agent — 와이어프레임 (레이아웃 설계)

> **이 문서는 코드가 아니라 "어떤 화면이 보일지"에 대한 승인용 스케치다.** 실제 구현
> (컴포넌트/차트 라이브러리)은 이 문서가 승인된 뒤 `04_frontend-chart-agent_notes.md`에서 정한다.
> 아래 모든 화면 요소는 `01_stock-data-agent_api-spec.md`(검색·OHLCV)와
> `02_technical-analysis-agent_output-schema.md`(지표·패턴·고급기법)의 **실제 응답 필드에만** 근거한다.
> 즉 화면에 존재하는 지표/패턴은 전부 백엔드가 실제로 계산해 주는 것들이고, 미구현 항목은
> "미구현" 상태로 명확히 표기한다.

핵심 설계 원칙 (chart-ui-design 스킬 기반):
- 지표/패턴/고급기법을 **3개의 토글 레이어**로 분리. 기본값은 "기본 지표만 ON".
- 검색은 한글/영문/티커 모두 지원 + 시장 배지(KOSPI/KOSDAQ/NYSE/NASDAQ)로 동명 종목 구분.
- 데이터 부족(`null`) 지표는 숨기지 않고 **회색 비활성 + 툴팁 사유**로 안내.
- 상승/하락은 색만이 아니라 **부호/아이콘 병기**(색맹 접근성).

## 1. 화면 A — 시작(검색) 화면

종목을 아직 안 고른 상태. EquiSense에서 검증된 "최근 검색 우선" 패턴을 재사용한다.

```
┌───────────────────────────────────────────────────────────────────────┐
│  Chart Deep Dive                                          [KR] [US]  ◑  │  ← 헤더: 로고 / 시장필터칩 / 다크모드
├───────────────────────────────────────────────────────────────────────┤
│                                                                         │
│                     차트를 파헤칠 종목을 검색하세요                          │
│                                                                         │
│        ┌─────────────────────────────────────────────────────┐         │
│        │ 🔍  삼성전자, AAPL, 005930 …                          │         │  ← 검색 입력(한글/영문/티커)
│        └─────────────────────────────────────────────────────┘         │
│                                                                         │
│        최근 검색                                                         │
│        ┌──────────────┐ ┌──────────────┐ ┌──────────────┐              │
│        │ 삼성전자       │ │ Apple        │ │ 카카오        │   ← 알약(칩), 클릭 시 바로 차트로
│        │ KOSPI 005930 │ │ NASDAQ AAPL  │ │ KOSPI 035720 │              │
│        └──────────────┘ └──────────────┘ └──────────────┘              │
│                                                                         │
└───────────────────────────────────────────────────────────────────────┘
```

- **최근 검색**: localStorage에 저장한 `{symbol, name, market, exchange}` 배열. 비어 있으면
  이 영역 자체를 숨긴다(초기 사용자).
- 헤더의 `[KR] [US]` 칩은 검색 결과를 시장으로 필터(둘 다 켜면 전체). 기본 둘 다 ON.

### 1-1. 자동완성 드롭다운 (입력 중)

`GET /api/search?q=<질의>` 응답의 `results[]`를 그대로 그린다. 각 행 = `name` + 시장배지(`exchange`) + `symbol`.

```
        ┌─────────────────────────────────────────────────────┐
        │ 🔍  삼성                                              │
        └─────────────────────────────────────────────────────┘
        ┌─────────────────────────────────────────────────────┐
        │  삼성전자              [KOSPI]            005930.KS  │  ← 불투명 배경(반투명 금지)
        │  삼성SDI               [KOSPI]            006400.KS  │
        │  삼성바이오로직스        [KOSPI]            207940.KS  │
        │ ─────────────────────────────────────────────────── │
        │  검색 결과가 더 없습니다                                │  ← results:[] 일 때 이 문구만
        └─────────────────────────────────────────────────────┘
```

- **시장 배지 색 구분**: KOSPI/KOSDAQ = 파랑 계열, NYSE/NASDAQ = 보라 계열. `exchange` 필드 그대로.
- 한글 질의는 Naver(KR)만, 영문/티커는 Yahoo+Naver 병합 — 이 차이는 사용자에게 안 보이고
  결과 목록으로만 자연스럽게 드러난다.
- 행 클릭 → 선택된 `symbol`+`market`으로 화면 B(차트)로 전환하며 최근검색에 저장.
- **에러 처리**:
  - `q` 공백/누락(400): 드롭다운을 아예 안 띄움(입력 유도).
  - 네트워크 실패: 드롭다운에 "검색을 불러오지 못했습니다. 다시 시도" 한 줄 + 재시도 버튼.
  - `results:[]`(정상 빈 결과): "일치하는 종목이 없습니다" — 에러 아님, 회색 안내.

## 3. 레이어 ① 기본 지표 (`indicators`) — 기본값 ON

| 지표 (응답 필드) | 표시 위치 | 표시 방식 |
|---|---|---|
| **이동평균 SMA/EMA** `indicators.sma.byPeriod[]`, `indicators.ema.byPeriod[]` | 메인 차트 위 오버레이 라인 | period별 라인(5·20·60·120). `values[i]`↔`dates[i]` 인덱스 정렬. 워밍업 `null`은 선 끊김. SMA/EMA 별도 토글, period별 색·범례. |
| **볼린저밴드** `indicators.bollinger.upper/middle/lower` | 메인 차트 위 밴드 | 상단·하단 라인 + 그 사이 옅은 음영, `middle`=SMA20 점선. |
| **RSI** `indicators.rsi.values` (0~100) | 메인 하단 **서브패널** | 라인 + 70/30 기준선. 현재값 숫자 표기. period(14) 라벨. |
| **MACD** `indicators.macd.macd/signal/histogram` | 메인 하단 **서브패널** | MACD·Signal 두 라인 + 히스토그램 바(양=빨강/음=파랑, 부호 병기). |

- 기본 지표 레이어를 켜도 **RSI/MACD 서브패널은 기본 접힘**(화면 과밀 방지). 체크 시 아래로 펼침.
- **MA period별 부분 null 처리**: `byPeriod`에는 데이터 충분한 period만 온다(예: 캔들 21개면
  60·120 빠짐). 그때 컨트롤의 `60`·`120` 체크박스는 **회색 비활성**, 툴팁:
  "캔들 60개가 필요합니다 (현재 21개)" — `meta.unavailable`의 사유를 그대로 노출.
- 지표 객체 자체가 `null`(예: RSI 캔들 15개 미만)이면 해당 체크박스 통째로 비활성 + 툴팁.

## 4. 레이어 ② 패턴 (`patterns`) — 기본값 OFF

`patterns.structural[]` + `patterns.harmonic[]`. 각 항목은 `range`(구간)와 `keyPoints`(근거 극값)를 가진다.

```
   메인 차트 위 오버레이 예시 (구조 패턴 = 이중바닥):

        90.1 ●─────────────────────● 90.6     ← keyPoints (kind:"bottom") 마커+점선 연결
             ╲                    ╱
   ┌ range.start ──────────── range.end ┐     ← 반투명 세로 밴드로 구간 하이라이트
   │  double-bottom · reversal · 0.97   │     ← 라벨칩: type · category · confidence
   └────────────────────────────────────┘
```

- **표시 규칙**: `range.start~end`를 메인 차트에 반투명 세로 밴드로 강조 + `keyPoints` 좌표에
  마커(도형)와 연결선. 라벨칩에 `type`·`category`·`confidence`(예: "이중바닥 0.97") 표기.
- **하모닉**은 XABCD 5점을 선으로 잇고 D를 "PRZ" 강조(응답 `note` 그대로 툴팁).
- 목록/필터: 컨트롤 패널에서 "구조 패턴"/"하모닉 패턴" 두 서브토글. 그 아래
  **탐지 목록**(confidence 내림차순 — 응답 정렬 그대로)을 리스트로 제공, 행 클릭 시 해당 `range`로
  차트 이동/포커스. 여러 패턴 동시 표시는 겹치면 지저분하므로 **리스트에서 선택한 것만 하이라이트**.

  ```
     ② 패턴  (탐지 4건)                 ← patterns 배열 총 개수
     ┌────────────────────────────────┐
     │ ● 이중바닥        reversal  0.97 │  ← 클릭=차트에 오버레이
     │ ○ 상승삼각수렴    continuation 0.81│
     │ ○ 갭상승(gap-up)  gap       0.66 │
     │ ○ Gartley(하모닉) harmonic  0.72 │
     └────────────────────────────────┘
  ```
- category별 색: reversal=주황, continuation=초록, gap=회청, harmonic=자주 (배지+아이콘 병기).
- **빈/부족 상태**: 캔들 25개 미만이라 `structural:[]`,`harmonic:[]`이고
  `unavailable["patterns"]`가 있으면, 목록 자리에 "패턴 탐지에는 최소 25개 캔들이 필요합니다" 회색 안내.
  탐지 0건(정상)이면 "탐지된 패턴이 없습니다".
- **미구현 패턴 안내**: 응답에 아예 안 오는 패턴(원형바닥/V자/깃발/컵앤핸들/채널/다이아몬드/섬반전)은
  화면에 넣지 않되, 패턴 패널 하단에 작은 "ⓘ 현재 탐지 범위" 링크 → 지원 패턴 목록을 안내(오해 방지).

## 5. 레이어 ③ 고급 기법 (`advanced`) — 기본값 OFF

| 기법 (응답 필드) | 표시 위치 | 표시 방식 |
|---|---|---|
| **일목균형표** `advanced.ichimoku` | 메인 차트 오버레이 | 전환/기준/후행선 라인 + **구름**(leadingSpanA/B 사이 음영). 구름은 `projectedDates`로 **오른쪽 26봉 미래 연장**(스키마 §5-1). A>B=빨강구름/A<B=파랑구름. `signal`(1/-1/0)을 요약칩("강매수/강매도/중립"). |
| **엘리엇 파동** `advanced.elliottWave` | 메인 차트 오버레이 | `impulse.waves[]`의 5개 `{label,date,price}`에 ①~⑤ 라벨 마커 + 파동 연결선. `direction`(up/down)·`completed` 표기, `signal` 요약칩. `impulse:null`이면 `reason` 문자열을 안내로 노출. |
| **변곡점 예측** `advanced.inflectionPoints` | — | **항상 `null`(백엔드 미구현)**. 컨트롤에서 `⊘` 영구 비활성 + 툴팁: "이 빌드에서는 제공되지 않습니다" (`meta.unavailable["advanced.inflectionPoints"]` 사유). |

- **일목 구름의 미래 연장**은 이 서비스의 시각적 하이라이트다. x축을 오늘 이후 26봉만큼
  여백으로 확보해 구름이 오른쪽으로 뻗는 걸 보여준다(`projectedDates` 사용).
- 일목/엘리엇을 동시에 켜면 매우 복잡 → 컨트롤에서 라디오처럼 "고급 기법은 한 번에 하나 권장"
  안내(강제는 아님). 켜지면 종목 요약바 우측 "신호요약"에 해당 `signal`을 뱃지로 노출.

### 5-1. 신호 요약 (종목 요약바 우측 "신호요약 →")

응답의 세 `signal`(ichimoku/elliottWave + 패턴 confidence 최상위)을 한 줄로 모아 보여주는 팝오버.
투자자문이 아니라 "탐지 결과 요약"임을 명시(면책 문구 1줄).

```
   신호요약
   ┌───────────────────────────────┐
   │ 일목균형표   ▲ 강매수 (+1)      │
   │ 엘리엇파동   ▼ 매도 (-1)        │
   │ 상위 패턴    이중바닥 0.97      │
   │ ── 참고용 요약이며 투자자문 아님 ──│
   └───────────────────────────────┘
```

## 6. 모바일 (반응형)

핵심(캔들+기본 지표)은 모바일에서도 보이고, 우측 레이어 패널은 **하단 시트/탭**으로 접는다.

```
┌───────────────────────────┐
│ ◀  삼성전자 005930.KS   ◑ │
│ ▲315,320 +2.31%           │
│ [1M 3M ●1Y 5Y]            │  ← 기간 가로 스크롤
├───────────────────────────┤
│                           │
│    메인 캔들 차트           │  ← 세로로 크게. MA/볼린저 오버레이 유지
│    (핀치 줌 / 좌우 스크롤)   │
│                           │
├───────────────────────────┤
│ [지표] [RSI] [MACD] [패턴] │  ← 서브패널을 탭 전환(동시 표시 X)
├───────────────────────────┤
│   (선택 탭 내용 영역)        │
├───────────────────────────┤
│  ⚙ 레이어 설정            ▲ │  ← 하단 시트: 3레이어 토글을 끌어올려 조작
└───────────────────────────┘
```

- 데스크톱의 우측 레이어 컨트롤 → 모바일에선 **하단 "레이어 설정" 시트**(끌어올림).
- RSI/MACD/패턴은 화면이 좁아 **탭 전환**(스킬 요구: "서브 패널은 모바일에서 탭으로 접거나 펼침").
- MA 오버레이는 유지하되 period 라벨은 탭 시 범례 팝업으로.

## 7. 로딩 / 에러 / 빈 상태 (전 화면 공통)

`/api/ohlcv`·`/api/analysis`는 공통 에러 봉투(`error.code`)를 준다. **404/502를 빈 데이터와 구분**한다(스펙 §6 요구).

| 상황 | 화면 |
|---|---|
| 로딩 중 | 차트 영역 스켈레톤 + "시세/분석 불러오는 중" |
| `404 NOT_FOUND` | "‘XXXX’ 종목의 데이터를 찾을 수 없습니다 (미상장/상장폐지 가능)". 검색으로 되돌리는 버튼. |
| `502 SOURCE_ERROR` | "데이터 소스 오류로 불러오지 못했습니다. 다시 시도" + 재시도. `cause`는 접이식 상세. |
| `400 BAD_REQUEST` | 잘못된 파라미터 — 개발 단계 노출용, 사용자에겐 "요청이 올바르지 않습니다". |
| 데이터 부족(200 + `null` 지표) | **에러 아님.** 차트는 정상, 해당 지표만 컨트롤에서 회색 비활성 + 툴팁 사유(`meta.unavailable`). |
| 15~20분 지연(KR) | 요약바에 작은 "지연 시세" 배지(스펙 §2-2 한계 명시). |

핵심: **데이터 부족은 조용히 숨기지 않는다.** 항상 "왜 이 지표가 없는지"를 툴팁으로 답한다.

## 8. 색상 / 접근성 규칙

- 상승/하락: 색(빨강/파랑) **+ 부호(▲▼, +/−)** 병기 → 색맹 사용자도 구분.
- 레이어 3색군: 기본=중립 회/청, 패턴=category별(주황/초록/회청/자주), 고급=일목 구름 빨/파.
- 다크·라이트 모드 모두 대응(헤더 ◑ 토글). 배지·툴팁 배경은 **불투명**(반투명 금지 — EquiSense 피드백).
- 차트 상 모든 오버레이는 범례로 이름·색 매핑 제공.
