# Chart UI Design

## 차트 라이브러리 선택 기준

범용 차트 라이브러리(recharts, chart.js)는 캔들스틱 + 다중 지표 오버레이 + 패턴 주석을 조합하기 번거롭다. 금융 차트 전용 라이브러리(예: TradingView의 `lightweight-charts`)를 우선 검토한다. 레이아웃 자체는 와이어프레임 단계(`_workspace/03_frontend-chart-agent_wireframe.md`)에서 먼저 정하고, 라이브러리 선택과 근거는 구현 단계의 `_workspace/04_frontend-chart-agent_notes.md`에 남긴다.

## 레이어 구조

지표가 많아질수록 화면이 복잡해지므로 세 층위를 토글 가능한 레이어로 분리한다:

1. **기본 지표 레이어**: 이동평균은 캔들 위에 라인으로, RSI/MACD는 메인 차트 하단의 별도 서브 패널로, 볼린저밴드는 캔들 위 밴드로 표시.
2. **패턴 레이어**: 헤드앤숄더/삼각수렴 등은 탐지된 가격 구간을 도형(추세선, 박스)으로 오버레이하고 `confidence`를 함께 표시.
3. **고급 기법 레이어**: 일목균형표는 구름(cloud) 형태로 메인 차트에, 엘리엇 파동은 파동 번호 라벨로, 변곡점 예측은 예측 시점 마커로 표시.

사용자가 한 번에 모든 레이어를 켜면 차트가 읽기 어려워지므로 기본값은 "기본 지표만 ON"으로 하고 나머지는 사용자가 선택적으로 켜게 한다.

## 검색 UX

- 종목명(한글/영문)과 티커 모두로 검색 가능해야 하며, 자동완성 결과에 시장 구분(KOSPI/KOSDAQ/NYSE/NASDAQ)을 함께 표시해 동명 종목 혼동을 막는다.
- 최근 검색 종목을 우선 노출하는 패턴은 이미 이 리포지토리의 다른 프로젝트(EquiSense)에서 검증된 UX이므로 참고할 만하다.

## 에러/빈 상태 표시

지표가 데이터 부족으로 `null`인 경우 UI에서 해당 지표를 회색 처리하고 이유를 툴팁으로 안내한다. 조용히 숨기지 않는다 — 사용자가 "왜 이 지표가 안 보이지"라고 헷갈리지 않게 한다.

## 접근성/반응형

- 차트는 데스크톱뿐 아니라 모바일 뷰포트에서도 핵심 캔들과 기본 지표는 보여야 한다. 서브 패널(RSI/MACD)은 모바일에서 탭 전환으로 접거나 펼 수 있게 한다.
- 색상은 상승/하락을 색맹 사용자도 구분 가능하게(색상만이 아니라 아이콘/부호 병기) 고려한다.
